<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → lang cookie | Toggle writes a low-trust UI-preference cookie (no auth value) |
| token declaration → rendered UI | Brand contract must render accessibly (colour never sole signal) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | spoofed/garbage `lang` cookie value | mitigate | `getLang()` accepts only exact `bg`, else falls back to EN default — no injection surface (Task 2) |
| T-04-02 | (Accessibility integrity) | status conveyed by colour alone | mitigate | StatusDot always renders a text label; Vitest asserts label presence per state (WCAG 1.4.1, Task 1) |
| T-04-03 | (i18n integrity) | missing BG translation shipping silently | mitigate | `bg: Dict` type annotation makes a missing key a `tsc` build failure (Task 2) |
| T-04-04 | Information Disclosure | lang cookie carrying anything sensitive | accept | Cookie holds only `en`/`bg`; no PII or auth value; not a meaningful target |
</threat_model>
