# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 1 builds the *minimal proof set* only (D-08) — design tokens, Montserrat, `StatusDot`, the 52px primary `Button`, and the EN/BG toggle control. It also stands up the admin sign-in screen and a near-empty post-login admin placeholder console. Inputs, cards, forms, and full layout shells are deferred to the phases that first need them (D-08, Deferred Ideas). This contract therefore declares the *whole brand system* (so later phases inherit it) but only Phase 1's surfaces consume it now.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside `StatusDot`, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, control inner padding |
| lg | 24px | Section padding, sign-in card padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical rhythm |

Exceptions:
- **Primary CTA height = 52px** (locked, brand) — a fixed component height, not a spacing token.
- **Minimum interactive hit target = 44px** (locked, brand) — applies to the EN/BG toggle and any tappable control; meets WCAG 2.5.5 / mobile-PWA touch sizing.

## Typography

Font family: **Montserrat**. Phase 1 declares 3 active roles + 1 reserved display role (≤4 total). Weights used in Phase 1: **regular 400** + **semibold 600** (2 weights, per contract). Heavier brand weights (700/800) are available in the loaded family but not introduced until a later phase needs them.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 600 (semibold) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Notes:
- Body 16px minimum prevents iOS Safari input zoom on the PWA.
- `StatusDot` text label uses the **Label** role (14px / 600), never colour alone (brand rule: status is always coloured dot + text label).

## Color

Brand palette: six colours + white (locked — source: PROJECT.md §Brand tokens). Mapped to the 60/30/10 model below for Phase 1's surfaces (admin sign-in + slate console chrome).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (`--white`) | Page background, primary surfaces, splash background |
| Secondary (30%) | `#2F4858` (`--slate`) | Admin console sidebar/dark surfaces, headings, body text |
| Accent (10%) | `#029B87` (`--teal`) | Primary CTA fill, links, focus rings, PWA theme/splash colour, `claimed` status |
| Destructive | `#E44B4B` (`--coral`) | Errors, alerts, unclaimed/cancelled status, destructive actions |

Full brand token set (declared in `@theme` now so all later phases inherit; semantic meanings are platform-wide):

| Token | Value | Semantic role |
|-------|-------|---------------|
| `--teal` | `#029B87` | primary / actions / links / `claimed` |
| `--teal2` | `#047982` | secondary / avatars |
| `--amber` | `#FEBE21` | in-progress / `en_route` / attention / live |
| `--coral` | `#E44B4B` | alerts / unclaimed / `cancelled` / destructive / errors |
| `--slate` | `#2F4858` | headings / text / dark surfaces / sidebar |
| `--grey` | `#66676F` | muted / `completed` |
| `--white` | `#FFFFFF` | base surface |

**Accent reserved for:** primary CTA fill, text links, input/control focus rings, the PWA theme/splash colour, and the `claimed` lifecycle status dot. NOT applied to all interactive elements — secondary/neutral controls use slate or grey.

**StatusDot lifecycle colour map** (defined here as the platform-wide contract; only proven visually in Phase 1, consumed fully from Phase 4 onward — every state pairs a coloured dot with a text label, never colour alone):

| State | Dot colour | Token |
|-------|-----------|-------|
| requested | grey | `--grey` |
| paid | teal2 | `--teal2` |
| claimed | teal | `--teal` |
| en_route | amber | `--amber` |
| arrived | amber | `--amber` |
| picked_up | teal | `--teal` |
| completed | grey | `--grey` |
| cancelled | coral | `--coral` |

Accessibility: every status is dot **plus** text label (colour is never the sole signal) — satisfies colour-blind / WCAG 1.4.1.

## Copywriting Contract

EN is the default; every string below has an EN and a BG entry in the typed dictionary (`en.ts` / `bg.ts`), keyed by string id (D-04/D-05). Copy shown here is the EN canonical; BG is authored alongside it. Phase 1 surfaces are the admin sign-in screen, the offline fallback page, and the placeholder console.

| Element | Copy (EN) |
|---------|------|
| Primary CTA | **"Send magic link"** (admin sign-in — passwordless, D-01) |
| Empty state heading | **"Nothing here yet"** (placeholder admin console — onboarding arrives in Phase 2) |
| Empty state body | **"Your console is ready. Companies, properties, and transfers will appear here as you set them up."** |
| Error state | **"We couldn't send your magic link. Check the email address and try again."** (sign-in failure → restate problem + next step) |
| Destructive confirmation | None in Phase 1. No destructive actions exist this phase (no records to delete/cancel/refund yet). Reserved pattern for later phases: `{action}: "This can't be undone. {consequence}. Continue?"` |

Additional Phase 1 strings (also dictionary-keyed, EN/BG):

| Element | Copy (EN) |
|---------|------|
| Magic-link confirmation | **"Check your email — we've sent you a sign-in link."** |
| Offline fallback heading | **"You're offline"** |
| Offline fallback body | **"Balkanity needs a connection for this page. We'll reconnect automatically when you're back online."** |
| Language toggle control | **"EN / BG"** (one-tap, persists via cookie; ≥44px hit target) |
| Sign-in email field label | **"Email address"** |
