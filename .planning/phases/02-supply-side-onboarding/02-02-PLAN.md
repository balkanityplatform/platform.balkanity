| Boundary | Description |
|----------|-------------|
| admin browser → RSC page | Authenticated request; role must be re-verified server-side before any read/render. |
| client form island → server action | Untrusted FormData crosses into a mutation; must be re-gated + zod-validated. |
| server action → Supabase (service-role) | Service-role bypasses RLS by design; the ONLY gate is `getCurrentRole()` in the action. |
| anon cookie-bound read → Postgres | Admin-read RLS policy is the data-layer boundary (defence-in-depth for reads). |
| local CLI → live Balkanity DB | Schema push must hit `qyhdogajtmnvxphrslwm` ONLY, never Kalvia. |
