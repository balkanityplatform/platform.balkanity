<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser → RSC page | Role re-verified server-side before read/render. |
| client form island → server action | Untrusted FormData (company_id, name) crosses into a mutation. |
| server action → Supabase (service-role) | Service-role bypasses RLS; the only gate is `getCurrentRole()` in the action. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-EOP3 | Elevation of Privilege | non-admin writing properties | mitigate | Server action re-gates via `getCurrentRole()`; admin-only RLS + no anon write policy (0002). |
| T-02-TMP3 | Tampering | forged/invalid company_id in FormData | mitigate | zod validates company_id as uuid; FK `references companies(id)` rejects a non-existent parent at the DB. |
| T-02-TMP4 | Tampering | deactivate-order bypass (D-12) | mitigate | "No active destinations" enforced inside `deactivateProperty` (`deactivatePropertyBlocked`) + FK `on delete restrict`; proven by `lifecycle.test.ts`. |
| T-02-ID2 | Information Disclosure | service-role key to client | mitigate | `createAdminClient` only in `"use server"` actions; `admin.ts` `import "server-only"`; never imported from PropertiesView/PropertyForm. |
| T-02-V5b | Tampering | malformed property input | mitigate | zod parse at the action boundary; generic dict-keyed `fieldRequired`/`saveFailed`. |
</threat_model>
