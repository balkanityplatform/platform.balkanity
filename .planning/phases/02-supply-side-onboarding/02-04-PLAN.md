The destinations vertical slice and the ONBD-06 acceptance bar. An admin can create, edit, list, and
soft-deactivate a bookable destination under a property — with the slug auto-filled from the label

onboarded through the UI with zero code/DB edits — the phase's definition of "no-code".

(second-company acceptance bar) all ship here. The slug uniqueness guarantee created here is the
surface Phase 4's `/pickup/<slug>` will consume.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser → RSC page | Role re-verified server-side before read/render. |
| client form island → server action | Untrusted FormData (slug, price, commission, parent id) crosses into a mutation. |
| server action → Supabase (service-role) | Service-role bypasses RLS; only gate is `getCurrentRole()`. |
| destination slug → future guest-facing /pickup URL | Slug is public; must be URL-safe and resolve to exactly one destination, exposing no PII (Phase 4 consumes it). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-EOP4 | Elevation of Privilege | non-admin writing destinations | mitigate | Server action re-gates via `getCurrentRole()`; admin-only RLS + no anon write policy (0002). |
| T-02-TMP5 | Tampering | duplicate/forged slug (collision race) | mitigate | DB `destinations_slug_key` unique index is the authority; `23505` caught → `t.slugTaken`; server-side slugify rejects non-URL-safe input (`slugInvalid`). |
| T-02-TMP6 | Tampering | out-of-range / float commission or negative price | mitigate | zod `commission_pct` 0–100 (`commissionRange`) + `price_cents` int >= 0; DB CHECK constraints (`commission_pct between 0 and 100`, `price_cents >= 0`) are the backstop; integer cents only (no float). |
| T-02-ID3 | Information Disclosure | slug leaking property/company PII | accept | Slug is admin-tuned, label-derived, contains no guest PII; /pickup PII gating is Phase 4/5 scope. Slug carries no property context in the URL (D-09). |
| T-02-ID4 | Information Disclosure | service-role key to client | mitigate | `createAdminClient` only in `"use server"` actions; `admin.ts` `import "server-only"`; never imported from DestinationForm/DestinationsView (the live "you keep" math uses the pure `commission` util, not the admin client). |
| T-02-V5c | Tampering | malformed destination input | mitigate | zod parse at the action boundary; generic dict-keyed `slugInvalid`/`commissionRange`/`fieldRequired`/`saveFailed`, no provider-detail leak. |
</threat_model>
