<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Stripe -> /api/stripe/webhook | Untrusted POST; authenticated only by HMAC signature, not a user session |
| guest browser -> /pay/success | Spoofable redirect target; must never be treated as payment confirmation |
| webhook handler -> wp_transfers (service-role) | The sole authoritative `paid` write path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-SPOOF1 | Spoofing | forged/unsigned webhook POST | mitigate | `stripe.webhooks.constructEvent` HMAC verification on the raw `req.text()` body; throw/missing-sig → 400 with zero state change (SC2; proven live in Plan 05). |
| T-03-SPOOF2 | Spoofing / Tampering | spoofed `success_url` confirms payment | mitigate | `app/pay/success/page.tsx` is display-only (reads status, never writes); `paid` set ONLY in the webhook (BOOK-05); enforced by the single-writer grep gate + the success-spoof e2e spec. |
| T-03-REPLAY | Tampering / Repudiation | replay of a captured event | mitigate | Insert-first into `webhook_events` with UNIQUE `event_id` + `ON CONFLICT DO NOTHING`; the `paid` UPDATE additionally carries `WHERE status<>'paid'` (SC3 — replay → exactly one effect). |
| T-03-LOG | Repudiation / Logging | unaudited events | mitigate | Every verified event is recorded in `webhook_events` with `event_id` + `signature_result` + `outcome` (received → processed / duplicate_skipped / no_matching_transfer) — HLTH-01. |
| T-03-IDLEAK | Information Disclosure | secret leak via wrong client | mitigate | Webhook + pages use the service-role client server-side only (no client component import of secrets); the route reads `STRIPE_WEBHOOK_SECRET` server-only. |
| T-03-START | Elevation of Privilege | unauthenticated charge trigger | mitigate | `app/pay/start` is admin/test-gated, not a public charge surface; Phase 4 replaces it with the booking form whose own validation gates the charge. |
</threat_model>
