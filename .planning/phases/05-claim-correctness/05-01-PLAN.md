## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test harness → live TEST DB | Service-role seed crosses into privileged writes; must stay TEST-DB-only and torn down |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | concurrency gate that secretly serializes (false-green) | mitigate | Single `Promise.all`, N independent caller clients, no `await`-in-loop; loop K times on reset rows (Pitfall 3) |
| T-05-02 | Information Disclosure | PII gate that checks UI instead of payload | mitigate | Assert raw `Object.keys(row)` on `wp_pool` + a base-table read returning 0 rows — payload-level, not UI |
| T-05-03 | Elevation of Privilege | service-role client leaking onto the claim path in tests | mitigate | Service-role confined to fixtures seed/teardown; `makeCallerClient` uses anon key + driver JWT (D-04) |
| T-05-04 | Spoofing | fixtures minting a winner via a client-supplied driver_id | mitigate | `claim_transfer` takes only `p_transfer_id`; the JWT (auth.uid) is the sole driver identity — gates never pass a driver id |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | Zero packages installed this phase (RESEARCH §Standard Stack — all deps already present); nothing to gate |
