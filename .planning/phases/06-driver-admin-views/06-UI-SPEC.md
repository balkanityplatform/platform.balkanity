# Phase 6 — UI Design Contract

> Visual and interaction contract for the Driver PWA surfaces (pool / claim / detail / "My run") and the Admin transfers console (list / detail / ops actions / manual refund). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase REUSES the locked Phase 1 design system — it does not introduce a new one.** Tokens (`app/globals.css` `@theme`), Montserrat, and the `platform/ui/` primitives (`Button`, `StatusDot`, `LifecycleTimeline`, `DataList`, `Card`, `Select`, `TextField`, `Toggle`, `LanguageToggle`) are the source of truth. Only two genuinely new presentational pieces are expected: a **toast** (lost-claim / action feedback) and a **refund form** (amount + reason + fee disclosure). Everything else composes existing primitives.

## Spacing Scale

Declared values (all multiples of 4 — matches the in-codebase convention exactly):

| Token | Value | Usage (as already used in `platform/ui/` + admin slices) |
|-------|-------|----------------------------------------------------------|
| xs | 4px | Dot↔label gap (`StatusDot` `gap-[4px]`), stacked metadata gap |
| sm | 8px | Compact element spacing, action-button clusters (`gap-[8px]`) |
| md | 16px | Default element spacing, list-row padding (`px-[16px] py-[12px]`), timeline row gap (`gap-[16px]`) |
| lg | 24px | Card padding (`Card` `p-[24px]`), header padding, CTA horizontal padding |
| xl | 32px | Section gaps (`gap-[32px]`) |
| 2xl | 48px | Page vertical rhythm (`py-[48px]`) |
| 3xl | 64px | Reserved for page-level breaks (use sparingly) |

Exceptions (brand / accessibility geometry — NOT spacing tokens, locked):
- **Primary CTA height: 52px** (`Button` `h-[52px]`) — the brand CTA height; the driver "advance status" CTA and the admin action buttons use this.
- **Minimum hit target: 44px** (`Button` `min-h-[44px]`) — every tappable control (claim button, run-card CTA, admin row actions, filter chips) must clear ≥44px (WCAG 2.5.5 / mobile PWA). List rows: `min-h-[56px]`.
- **Status dot: 10px** (`h-[10px] w-[10px]`) — fixed brand indicator size, not on the spacing scale.

## Typography

Exactly four roles, two weights in active use (400 regular + 600 semibold). Montserrat throughout. These values are read verbatim from the shipped components — do not introduce new sizes.

| Role | Size | Weight | Line Height | Where (existing usage) |
|------|------|--------|-------------|------------------------|
| Display (page title) | 28px | 600 (semibold) | 1.2 | `h1` page title (`text-[28px] font-semibold leading-[1.2]`) |
| Heading (section) | 20px | 600 (semibold) | 1.3 | `h2` section heads (`text-[20px] font-semibold leading-[1.3]`) |
| Body | 16px | 400 regular (600 for primary names/CTAs) | 1.5 | Body copy, list primary name, CTA label (`text-[16px]`) |
| Label / meta | 14px | 600 (status labels) / 400 (muted meta) | 1.4 | `StatusDot` label, row metadata, timeline timestamps (`text-[14px]`) |

Weight rule: **400 and 600 only.** 500/700/800 exist in the font stack but are NOT used in this phase's surfaces (keeps the two-weight contract). Status labels and primary names are 600; everything else is 400.

## Color

Brand palette is the locked six-colour + white set from `app/globals.css` `@theme`. The 60/30/10 split below describes *this phase's* surfaces; accent colours are reserved per the lifecycle/semantic map and must never be used decoratively.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--white #FFFFFF` | Page + card surfaces (both driver and admin content area) |
| Secondary (30%) | `--slate #2F4858` (admin chrome) / `--grey #66676F` (muted text, completed) | Admin header/sidebar chrome; muted metadata + completed-state text across both surfaces |
| Accent (10%) | `--teal #029B87` | See reserved-for list |
| Destructive | `--coral #E44B4B` | See reserved-for list |

**Accent (`--teal`) reserved for** — never "all interactive elements":
- Primary CTAs only (`Button` primary: claim, advance-status, admin save/confirm actions, refund-confirm).
- The `claimed` and `picked_up` lifecycle dots (per `StatusDot` map, locked).
- Active/affirmative `DataList` indicator dot, links, focus ring (`outline-teal`).
- `--teal2 #047982` is the secondary accent: `paid` lifecycle dot + avatars only.

**Semantic colours (status-driven, never decorative) — locked `StatusDot` / brand map:**
- `--amber #FEBE21` → `en_route` + `arrived` lifecycle dots; "in-progress / attention / live" cues.
- `--coral #E44B4B` → **destructive + alert only**: `cancelled` dot, unclaimed/"needs-attention" highlight rows in the admin list (D-07/D-09 — coral pinned-top rows), the Cancel/Refund/Release confirm-dialog destructive button, error toasts/messages.
- `--grey #66676F` → `requested` + `completed` dots, muted metadata, the "Completed today" collapsed driver section (D-06).

**Color-is-never-the-sole-signal (WCAG 1.4.1, brand-locked):** every status is a **coloured dot + text label** (`StatusDot`). The admin "needs attention" coral rows MUST also carry a text marker (e.g. an "Unclaimed" / "Stuck" label or coral-dotted status), never coral background alone.

## Interaction Contracts (phase-specific behaviour the executor must honour)

| Contract | Rule | Source |
|----------|------|--------|
| Pool "live" refresh | Refetch-on-focus + light interval poll (~20–30s, planner discretion). NOT Supabase Realtime. NOT SW-cached — NetworkFirst always. | D-01, CONTEXT |
| Winning claim | Render driver detail (full PII) **from the `claim_transfer()` returned row** — no follow-up fetch (no read-then-write on the claim path). | D-02, Phase 5 |
| Losing claim | Neutral toast + card silently leaves the pool. Expected outcome, never an error state. | D-03 |
| Advance status | Inline single next-step 52px CTA on the run card; next edge resolved via `lifecycle.ts` `ALLOWED_TRANSITIONS`. Compact read-only `LifecycleTimeline` beside it. | D-04, D-05 |
| Completed | Drops out of active run into collapsed "Completed today" section; active run stays ordered by arrival time; no un-claim. | D-06, CLAIM-04/06 |
| Admin default sort | Soonest arrival first; coral "needs attention" rows pinned to top. | D-07 |
| Stuck highlight | UI highlight only (no alerts — Phase 8): unclaimed always coral; near-arrival-unclaimed and arrived-stalled coral via simple pilot constants. | D-09 |
| Refund hook | NEW server-only `platform/payments/*` hook calling `stripe.refunds.create`; never client-side; never sets `paid`; reuses `fee.ts` for the disclosure figure. | D-12, OPS-04 |
| Offline | No offline claim/advance (conflicts with atomic-claim guarantee). Run is NetworkFirst read-only at most. | CONTEXT |
