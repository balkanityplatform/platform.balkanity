<objective>
Deliver the in-app notification bell + feed — the user-observable NOTF-01 slice and the drivers' primary channel. A driver or admin sees an "Alerts" trigger with an unread count in their header chrome; opening the feed shows their own notifications newest-first (unread = teal dot + 600 title; cap-near = coral dot + text), marks items read on open, and offers "Mark all read". The bell polls on focus + visibilitychange + a ~25s interval (copying the proven PoolView shape — NOT Realtime, D-04).

Purpose: After this plan, a real user can open their bell and see/clear alerts live. It consumes the engine's `markRead`/`markAllRead` (Plan 02) and the `notifications` table (Plan 01). The fan-out that PRODUCES notifications is Plan 04 — but the read/clear surface is independently testable against seeded rows.
Output: feed read, bell island, the bell mounted in both driver (warm-light) and admin (slate) chromes, and the notifications-rls spec GREEN.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| driver/admin browser → bell RSC seed | role-gated server read; caller-auth RLS scopes to own rows |
| client poll → refetchNotifications | same own-rows-only read as the RSC seed — the poll cannot widen the surface |
| client → markRead/markAllRead | caller identity (auth.uid()) is the recipient-scope gate; no client write RLS policy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-BELL1 | Information Disclosure | bell read widening to other users' notifications | mitigate | feed.ts uses the caller-auth client (never service-role); RLS notifications_own_read scopes to recipient_id = auth.uid(); refetch reuses the same read |
| T-07-BELL2 | Spoofing/Elevation | non-driver/non-admin reaching the bell surface | mitigate | RSC pages re-gate with getCurrentRole() server-side (never getSession); guests get no bell (D-01) |
| T-07-BELL3 | Tampering | client marking another user's notifications read | mitigate | markRead/markAllRead scope every UPDATE to recipient_id = auth.uid() (gated service-role action); no client write RLS policy |
| T-07-BELL4 | (UX-integrity) | stale unread state served from SW cache | mitigate | bell data path is live (poll-on-focus); NetworkFirst at most — never SW-cached stale (mirrors Phase 6 pool data-path rule) |
</threat_model>
