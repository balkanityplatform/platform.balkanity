## Spacing Scale

The "Balkanity Path" strict **8px grid** (DESIGN.md "Layout & Spacing"). Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (StatusDot dot↔label), inline padding |
| sm | 8px | Compact element spacing, standard corner radius |
| md | 16px | Default element spacing; mobile safe-area margins; card gutter |
| lg | 24px | Section padding; button horizontal padding (`px-[24px]`) |
| xl | 32px | Layout gaps; tablet/desktop margins |
| 2xl | 48px | Major section breaks; **input field height (48px)** |
| 3xl | 64px | Page-level spacing |

Exceptions (brand-locked, all ≥ multiple-of-4):
- **44px** minimum touch target on every interactive element (DESIGN.md; WCAG 2.5.5; `--spacing-touch-target`).
- **52px** primary CTA height (DESIGN.md; existing `Button` `h-[52px]`; `--spacing-cta-height`).
- **10px** status dot diameter (DESIGN.md "Status Indicators" — a 10px coloured dot; existing `StatusDot` `h-[10px] w-[10px]`). Decorative size only, not a layout-grid unit.

Margins: 16px on mobile, 32px on tablet/desktop (DESIGN.md).

## Typography

Montserrat exclusively. The "Balkanity Path" scale, mapped to the template's four roles plus the brand's full named scale (executors use the named roles). **Two weights are the working contract** (400 regular + 600 semibold — the weights actually loaded in `app/layout.tsx`). DESIGN.md also names 700 (bold) and 500 (medium); per the GSD 2-weight discipline, **regular (400) + semibold (600) are the declared pair** — 700 may be used ONLY for the single display/headline-xl role if a heavier headline is required, and 500 maps to 600 in implementation (do not load extra weights).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 (24px) |
| Label | 14px | 600 | 1.4 (≈20px) |
| Heading | 24px | 600 | 1.33 (32px) |
| Display | 28px (mobile) / 32px (desktop) | 600 (700 permitted for this role only) | 1.2 (34px / 40px) |

Full "Balkanity Path" named scale (for executor reference — implement with the 400/600 pair):

| Named role | Size | Weight | Line height |
|------------|------|--------|-------------|
| headline-xl | 40px | 600/700 | 48px |
| headline-lg | 32px | 600/700 | 40px |
| headline-lg-mobile | 28px | 600/700 | 34px |
| headline-md | 24px | 600 | 32px |
| body-lg | 18px | 400 | 28px |
| body-md (Body) | 16px | 400 | 24px |
| body-sm | 14px | 400 | 20px |
| label-lg (Label) | 14px | 600 | 16px |
| label-md | 12px | 600 (was 500) | 16px |

Body line-height 1.5; heading line-height ≤1.2 on display sizes (DESIGN.md letter-spacing −0.01 to −0.02em on headlines).

## Color

The "Balkanity Path" palette is **functional/semantic** — colour communicates transfer status. Surfaces differ per surface-phase: Guest/Driver = warm light (white base); Admin console = slate. The 60/30/10 split below is the platform default; the Admin slate sidebar is the documented 30% secondary surface.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` white | Base surface / page background (DESIGN.md "Base Layer: Pure White"); card backgrounds |
| Secondary (30%) | `#2F4858` slate | Headings, body text, dark surfaces, the Admin left sidebar / console chrome |
| Accent (10%) | `#029B87` teal | See reserved-for list below |
| Destructive | `#E44B4B` coral | Alerts, errors, Unclaimed + Cancelled status, destructive actions only |

Accent (`#029B87` teal) reserved for: primary CTA fill (`Button` primary), active navigation item, links, focus ring (2px teal on inputs + `focus-visible` outline), the **Claimed** status colour, and the Transfer Badge background. NOT for arbitrary decoration or every interactive element.

Full semantic token map (already in `app/globals.css` `@theme`; this phase confirms + extends, does NOT redefine the primary):

| Token | Hex | Semantic role |
|-------|-----|---------------|
| `--color-teal` | `#029B87` | Primary / actions / links / **Claimed** / active nav |
| `--color-teal2` | `#047982` | Deep teal — supporting brand, avatars, **Paid**, secondary-button stroke |
| `--color-amber` | `#FEBE21` | Live attention — **En route** / Arrived / in-progress |
| `--color-coral` | `#E44B4B` | Alerts / errors / **Unclaimed** / **Cancelled** (hollow ring) / destructive |
| `--color-slate` | `#2F4858` | Headings / text / dark surfaces / Admin sidebar |
| `--color-grey` | `#66676F` | Muted / Requested / **Completed** / borders / secondary text |
| `--color-white` | `#FFFFFF` | Base surface |

**Status colour mapping (DS-02 — coloured dot/badge PLUS worded label, NEVER colour alone, WCAG 1.4.1):**

| Status | Treatment | Worded label |
|--------|-----------|--------------|
| Unclaimed | Coral solid dot/badge | "Unclaimed" |
| Claimed | Teal solid dot/badge | "Claimed" |
| En route | Amber solid dot/badge | "En route" |
| Completed | Grey solid dot/badge | "Completed" |
| Cancelled | Hollow coral ring (2px stroke, transparent fill) | "Cancelled" |

Note for executor: the existing `StatusDot` covers all 8 lifecycle states (requested/paid/claimed/en_route/arrived/picked_up/completed/cancelled) with solid dots. DS-02 requires the **cancelled** treatment to become a **hollow coral ring** (not a solid coral dot) — this is the one status-badge change this phase introduces. Keep the always-present worded label.

**Tonal elevation, not heavy shadows** (DESIGN.md): content cards use a 1px border (`#66676F` @ 20% opacity) OR a soft ambient shadow `0px 4px 12px rgba(47,72,88,0.08)`. Overlays/bottom sheets use a slate-navy backdrop at 40% opacity. Buttons are solid fills, no gradients.

**Radii** (DESIGN.md "Shapes" — to add to `@theme` as `--radius-*`): sm 4px (0.25rem), DEFAULT/standard 8px (0.5rem — inputs, cards, small buttons), md 12px (0.75rem), lg 16px (1rem — containers, bottom sheets), xl 24px (1.5rem), full 9999px (Transfer Badge circle, pills).

## Copywriting Contract

Phase 9 ships **components**, not full screens, so copy is the component-level and shared-state copy these primitives carry. All user-facing strings live behind the EN/BG dictionary parity gate (`platform/i18n` en.ts/bg.ts — tsc fails on a missing key; this is the established project pattern). Status labels are the worded labels in the Status table above.

| Element | Copy |
|---------|------|
| Primary CTA (default label on `Button` primary) | Context-supplied by consuming surface (e.g. Guest "Pay & confirm transfer", Driver "Claim transfer", "Confirm arrival"). Phase 9 ships the styled CTA shell; no hardcoded verb. |
| Empty state heading | "Nothing here yet" (shared `EmptyState` primitive heading; surfaces override) |
| Empty state body | "New transfers will appear here as they come in." (surfaces override with surface-specific next step, e.g. driver pool, admin list) |
| Error state | "Something went wrong. Please try again." — shared inline error copy; pairs problem + retry path. Field-level validation copy is owned by the existing form primitives (unchanged this phase). |
| Destructive confirmation | None in Phase 9. No destructive action is wired in the foundation phase — Cancelled is a **rendered status** (hollow coral ring + label), not an action. Destructive-action confirmations (cancel/refund) are owned by the Admin phase (12). |

Stepper labels (DS-04 lifecycle stepper, exact order + casing): **Paid → Claimed → En route → Arrived → Picked up → Completed**. These are the worded step labels; the stepper also encodes completed/active/pending via shape+colour (not colour alone).
