## Trust Boundaries

|----------|-------------|
| driver browser → signOutAction | New auth/session-only server action: clears the @supabase/ssr session cookie + redirects; no payload, no DB |
| driver → auth.getUser() identity | Already-verified session (revalidated JWT); identity header is display of the caller's own email/name |
| driver → saveDigestPreference (unchanged) | Narrow service-role write keyed to verified auth.uid() (driver_profiles has no self-write RLS) |

The sign-out action is the ONE new write in the whole phase. It is auth/session-only: `supabase.auth.signOut()` clears the cookie and redirects — no schema, no RLS, no table, no payload. It does NOT touch payment/claim/PII paths, so it does not trip the schema/auth review gate beyond a normal sign-out (D-05, RESEARCH Security verdict).
