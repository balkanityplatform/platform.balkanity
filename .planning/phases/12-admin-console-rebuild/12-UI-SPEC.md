# Phase 12 — UI Design Contract

> Visual and interaction contract for the **Admin Console Rebuild** — the final surface of the v1.1 UI Rebuild milestone (Guest → Driver → Admin). The admin desktop console is re-skinned to the **"Transfer Pool"** identity: a persistent left sidebar (Dashboard / Transfers / Drivers / Settings), KPI cards computed from real transfer data, the pending-transmissions transfers table with filter + sort and row actions, a restyled transfer detail keeping every existing operation intact, and a top bar with client-side search, the notifications bell, and the signed-in admin identity. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope (presentation-only):** This phase re-skins the EXISTING admin routes — `app/admin/page.tsx` (landing → Dashboard), `app/admin/transfers/page.tsx` (+ `TransfersView.tsx`), `app/admin/transfers/[id]/page.tsx` (+ `TransferDetailView.tsx`, `RefundForm.tsx`), `app/admin/drivers/`, and the supply CRUD surfaces (`companies/`, `properties/`, `destinations/`) + `health/` that the sidebar reaches. **NO** change to backend, schema, auth, RLS, payment path, the unmasked `wp_transfers_admin_read` RLS read, the five gated ops Server Actions (`assign` / `reassign` / `release` / `cancel` / `refund`), the `refundPayment` hook, or the single-writer `paid` invariant. KPI cards and the transfers table read the EXISTING admin transfer query (anon cookie-bound client, RLS-gated). The top-bar search is a **client-side filter of already-loaded transfers** (no new endpoint). The notifications bell reuses the EXISTING `NotificationBell` + own-rows `readOwnNotifications()` feed. The refund still records `last_action_*` only and **NEVER sets `paid`** (D-12). The role gate (`getCurrentRole() === "admin"`) runs server-side BEFORE any read on every route — unchanged.
>
> **Foundation:** This surface **consumes** the Phase 9 design system and follows the Phase 10/11 surface-local pattern — it does NOT re-invent tokens or shared components. All tokens already live in `app/globals.css` `@theme`; the shared components ship from `platform/ui/` (`StatusDot` + `stateLabel`, `RouteMotif`, `LifecycleStepper`, `NotificationBell`, `LanguageToggle`, `Button`). New admin-only chrome (sidebar, top bar, KPI card, table) lives under `app/admin/` (mirroring `app/driver/_nav/` and `app/(guest)/_pass/`), **NOT** in `platform/ui/`. The console is the **slate** surface variant (slate sidebar + chrome) — distinct from the Guest/Driver warm-light variant.
>
> **Source of truth:** the Stitch admin mockups (Transfer Pool dashboard / transfers table / transfer detail) + `Branding/.../balkanity_path/DESIGN.md`, **corrected** on contact. Brand primary is **`#029B87`** — the mockups' `#00685a`/`primary` token family is **rejected**. Material Symbols icons → Phase 9/10/11 line pictograms + the committed brand Transfer Badge. The locked decisions below are fixed.

## Locked Decisions (this phase)

These decisions are driven by the ROADMAP Phase 12 contract + the v1.1 truthfulness rule + the Guest/Driver precedents, and are FIXED — do not re-litigate.

| # | Decision | Contract |
|---|----------|----------|
| 1 | **Omit backend-less mockup features.** | DROP the **"Analytics" nav page** (no analytics backend — the sidebar has NO Analytics item), the **"Download Manifest" export** (no export endpoint), and any **invented KPI "daily-goal %"** progress metric. None have backing data (ROADMAP Out of Scope / REQUIREMENTS). The dashboard shows ONLY the four data-true KPI cards below. |
| 2 | **Sidebar = Dashboard / Transfers / Drivers / Settings (4 items, no Analytics).** | A persistent left sidebar (slate) replaces the current top-bar-only section nav. Four items, each a line-icon + label, active item highlighted in teal: **Dashboard** (`/admin`), **Transfers** (`/admin/transfers`), **Drivers** (`/admin/drivers`), **Settings**. "Settings" **aggregates the existing supply CRUD + health** — it links to a settings hub (or directly groups Companies `/admin/companies`, Properties `/admin/properties`, Destinations `/admin/destinations`, Platform health `/admin/health`). No new routes are invented; Settings is a grouping of routes that already exist. Active highlight is `usePathname`-driven (prefix match keeps the parent lit on child routes, mirroring `DriverBottomNav`). |
| 3 | **KPI cards = Unclaimed / Claimed / En route / Total today, from real transfer data.** | The Dashboard shows exactly four KPI cards, each a **count derived from the already-loaded transfer rows** (the existing admin read): **Unclaimed** = `status='paid' AND driver_id IS NULL` (coral accent), **Claimed** = `status='claimed'` (teal accent), **En route** = `status='en_route'` (amber accent), **Total today** = transfers whose `arrival_at` (or paid/created date — implementer picks the one already available on the row) falls on the current local day (slate/neutral accent). NO daily-goal %, NO trend arrows, NO earnings (Decision 1). Each card = big figure (Display) + worded label (Label) + status-coloured accent (colour is never the sole cue — the worded label is always present, WCAG 1.4.1). |
| 4 | **Transfers list = the pending-transmissions table with filter + sort + row actions.** | The transfers list is restyled from the current mobile-style stacked `<ul>` into the **"pending-transmissions" table** (semantic `<table>`): columns **Time / ID** (arrival date·time + truncated transfer ID), **Passenger** (guest name), **Route** (airport → zone, compact), **Lifecycle** (inline `LifecycleStepper`-style mini bar OR `StatusDot` progression), **Status** (`StatusDot` pill + worded label), **Assigned driver** (name or "—" / "Unassigned"), **Actions** (row actions: **View** → detail, **Assign**, **Cancel**). The existing status filter chips + needs-attention quick filter are preserved (and remain server-re-query via URL params — unchanged). **Sort** is added as a presentation control over the loaded rows (by time / status / attention). The coral needs-attention left-border + text badge are preserved (WCAG 1.4.1). |
| 5 | **Top-bar search is a CLIENT-side filter of loaded transfers (AUI-05).** | The top bar carries a **search field that filters the already-loaded transfer rows client-side** (guest name / flight no. / destination / ID) — NO new endpoint, NO server round-trip for this control. (The existing URL-param `q` server search may remain as-is on the Transfers page; the AUI-05 top-bar search is the new client-side filter over the loaded set, satisfying "client-side filter of loaded transfers".) The top bar also carries the **notifications bell** (existing `NotificationBell` + `readOwnNotifications()` feed, mounted once in the admin shell) and the **signed-in admin identity** (admin email/name, read from the existing session — no new auth call). The `LanguageToggle` stays in the top bar. |
| 6 | **All five existing ops are preserved verbatim, restyled only.** | The transfer detail keeps `assign` (one-tap inline driver-id form), `reassign` (reason dialog + driver-id), `release` (reason dialog), `cancel` (reason dialog, destructive, offers the refund shortcut — never auto-refunds, D-11), and `refund` (`RefundForm`, full-amount pre-fill editable down, always-shown fee disclosure, records `last_action_*` only, **NEVER sets `paid`**, D-12). The vertical `LifecycleTimeline` on the detail is **swapped for the horizontal `LifecycleStepper current={status}`** (DS-04) to match the Guest/Driver precedent. No new action, no changed action behaviour, no changed authz. |

**Presentation-only omissions (ROADMAP v1.1 truthfulness rule):** Analytics nav page + any analytics charts, "Download Manifest" / CSV export, KPI daily-goal % progress ring, trend arrows (+12% etc.), revenue/earnings totals, driver ratings, live GPS map. None have backing data — all omitted.

## Spacing Scale

Consumes the Phase 9 8px grid unchanged. All multiples of 4. This is a **desktop console** surface (sidebar + content area), so the spacing leans to the larger end of the scale vs the mobile Guest/Driver surfaces.

| Token | Value | Usage on this surface |
|-------|-------|-----------------------|
| xs | 4px | Icon↔label gaps (StatusDot dot↔label, KPI label↔figure, nav-icon↔label) |
| sm | 8px | Compact stacking; table cell inner padding; standard 8px corner radius; chip gap |
| md | 16px | Table cell horizontal padding; card padding; KPI-card grid gap; top-bar control gap |
| lg | 24px | Content-area page padding; section padding; sidebar item vertical rhythm |
| xl | 32px | Desktop content margin; gap between dashboard sections (KPI row ↔ table) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top spacing on wide viewports |

Exceptions (brand-locked, all ≥ multiple-of-4):
- **44px** minimum touch/hit target on every interactive element — sidebar items, row actions, filter chips, top-bar controls, KPI cards if linked. WCAG 2.5.5; `--spacing-touch-target`.
- **52px** primary CTA height (`Button` `h-cta-height`; `--spacing-cta-height`) — the detail-page ops CTAs (assign/confirm dialogs) stay at 52px per the existing implementation.
- **10px** status dot diameter (decorative; from `StatusDot`).
- **Sidebar width ≈ 240–256px** fixed on desktop; the content area is the remaining width. **Top bar height ≈ 64px**. Table rows ≥ 56px (existing `min-h-[56px]` row preserved as the table row height floor) so each row is a comfortable hit area.
- Content max-width may relax from the current mobile-ish `max-w-2xl` to a **wider console content column** (e.g. `max-w-6xl` / full content area) appropriate to a desktop table — the console is desktop-first.

Margins: 24–32px content-area gutter on desktop (existing `px-[24px]` retained as the floor).

## Typography

Montserrat exclusively, **400 + 600 pair** (the loaded weights). Phase 9 roles, consumed verbatim. The mockups' 700/800 and 40px display are NOT introduced.

| Role | Size | Weight | Line Height | Used for |
|------|------|--------|-------------|----------|
| Display | 28px | 600 | 1.2 (34px) | Page title ("Transfer Pool" / "Transfers" / detail guest name); the **KPI figure** (the big count) |
| Heading | 24px | 600 | 1.33 (32px) | Section headings ("Recent transfers"), settings-hub group titles, empty-state heading |
| Body | 16px | 400 | 1.5 (24px) | Table cell primary values (passenger, route, driver), detail fact values, dialog prompts |
| Label | 14px | 600 | 1.4 (≈20px) | Table column headers, KPI card label, status labels, sidebar item labels, filter chips, fact captions (UPPERCASE permitted on column headers + KPI labels) |

Caption micro-text from the mockup (`text-[10px]`/`text-[11px]`) is **lifted to 14px Label** (12px `label-md` is the absolute floor; the existing coral `needsAttentionBadge` at 12px/600 is the one accepted 12px instance, carrying a redundant text label so it is never colour-alone). **Do not ship sub-14px body text.** UPPERCASE tracking is permitted on table column headers ("TIME / ID", "PASSENGER", "ROUTE", "STATUS", "DRIVER", "ACTIONS"), KPI labels ("UNCLAIMED", "CLAIMED", "EN ROUTE", "TOTAL TODAY"), and the needs-attention badge as a styling treatment of the Label role. The signed-in admin identity in the top bar renders at Body/Label (14–16px), never below 14px.

## Color

The admin console is the **slate** surface variant (the documented 30% secondary surface used as the console chrome). The sidebar + top bar are slate; the content area is white. This is the inverse emphasis of the Guest/Driver warm-light surfaces and is intentional (ROADMAP: "Slate console surfaces").

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` white | Content-area background; KPI card background; table background; detail card background |
| Secondary (30%) | `#2F4858` slate | The persistent left **sidebar** + the **top bar** chrome; headings/body text on white; table column-header text |
| Accent (10%) | `#029B87` teal | See reserved-for list below |
| Destructive | `#E44B4B` coral | Unclaimed KPI accent, needs-attention row border + badge, Cancelled status (hollow ring), cancel/refund destructive ops controls, claim/ops error toast |

**Accent (`#029B87` teal) reserved for:** the **active sidebar item** (teal icon + label + indicator on the slate sidebar), the **primary ops CTAs** (assign / confirm-action fill), the **Claimed** status colour + the **Claimed KPI** accent, the `LifecycleStepper` **completed-step** teal circle + connector, the `RouteMotif` **Transfer Badge** circle, **row "View" / detail links** (teal), the top-bar **search focus ring** + input `focus-visible` outline, and the logo chip. NOT for arbitrary decoration, not on every interactive element, not on the destructive ops (those are coral).

**Status / lifecycle colours** are owned by the shared components (do NOT re-map):
- Transfers-table **Status** column uses `StatusDot` for each row's real state: Unclaimed (`paid` + no driver) presented as the DS-02 **coral "Unclaimed"** badge, Claimed=teal, En route=amber, Completed=grey, Cancelled=hollow coral ring — always coloured dot/badge **+ worded label** (WCAG 1.4.1). (Presentation choice over the same unmasked admin data, mirroring the Driver pool's Unclaimed=coral mapping — no data change.)
- The transfers-table **Lifecycle** cell and the detail page render the lifecycle via `LifecycleStepper` (completed=teal+check / active=amber / pending=grey outline ring) — shape-encoded, never colour alone; each step keeps its worded label.
- **KPI card accents** (Unclaimed=coral, Claimed=teal, En route=amber, Total today=slate/neutral) are a coloured accent **plus** the always-present worded label — colour is never the sole cue (WCAG 1.4.1).

**Elevation:** tonal, not heavy (DESIGN.md). KPI cards + content cards keep a 1px `#66676F`@~20–30% border (`border-grey/20`–`/30`) + soft `shadow-sm` (the Phase 9 ambient `0 4px 12px rgba(47,72,88,0.08)` is the heavier permitted option for a hero/detail card). The table uses 1px `divide-grey/20` row separators (existing pattern). The sidebar gets a 1px right border / the top bar a 1px bottom border (`border-grey/20` on the white seam, or a darker slate seam within the chrome). Buttons are solid fills, no gradients.

**Radii** (from `@theme`): KPI cards + content/detail cards `rounded-md` (12px) — matching existing admin cards; the table container `rounded-md`; CTAs `rounded-md`/`rounded-lg` (existing `Button`); status dots + Transfer Badge `rounded-full`; the needs-attention pill badge `rounded-[4px]` (existing). Sidebar active-item indicator may use `rounded-md`.
